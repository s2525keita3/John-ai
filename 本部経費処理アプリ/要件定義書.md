# 本部経費処理アプリ — 要件定義（蓄積版）

**文書の位置づけ**  
原本は `本部経費処理 自動化 要件定義書 v2.docx`（ユーザードキュメント）。本書は **2026年4月時点** の実装（MVP）に対応した要件の整理・追記である。

---

## 1. 目的

- 月次の本部経費を **自社PL勘定項目** に振り分け、レビュー可能な形で集計する。
- **誤仕訳を防ぐ**（特にスキャンOCR・不明行の明示）。
- 手打ちPL運用ルールを **マスタとして蓄積**し、キーワード・金額レンジで再利用する。

---

## 2. スコープ（実装済み）

| 区分 | 内容 |
|------|------|
| 取込 | 複数金融・カードフォーマット（CSV／PDF） |
| 振分 | 摘要キーワード（部分一致）＋任意の金額下限・上限＋任意のデータソース区分 |
| 分類結果 | 確定／要確認／判断不能（マスタ照合結果） |
| 人件費 | 支給控除一覧（部門別）から本部対象者の人件費合計を集計 |
| 除外 | 摘要に「オリコ」を含む行の除外（任意） |

---

## 3. 機能要件

### 3.1 取引データ取込

| ID | 要件 |
|----|------|
| F-01 | **あおぞらネット銀行** 標準CSV（列: 日付・摘要・入金金額・出金金額・残高・メモ）。文字コードは utf-8-sig / cp932 等を自動試行。 |
| F-02 | **アメックス activity CSV**（ご利用日・ご利用内容・金額）。金額はカンマ付き文字列を数値化。マイナス金額は集計上0扱い。 |
| F-03 | **横浜信用金庫** 入出金明細 **CSV**（日付・科目・支払先・摘要・入金・出金・計）。科目＋支払先＋摘要を結合して照合テキストとする。 |
| F-04 | **横浜信用金庫** **通帳スキャンPDF** を OCR で読み取る。テキストレイヤが十分な場合はそれを利用。画像のみの場合は Tesseract または EasyOCR。行ごとに **取込ステータス**（確定／要確認／不明）を付与し、**誤取り込み防止**のため既定では確定行のみ振分対象とする（要確認行の含否はオプション）。 |
| F-05 | **エネクスフリート（エネオス）** 請求書 **PDF**。テキスト抽出し、**（車番　計）** 行の金額を **カード番号 0001〜0004**（本部想定）ごとに1行取込（給油明細の行単位の合計は使わない）。 |
| F-06 | **手動**で列名を指定して任意CSVを読む。 |

### 3.2 振り分けマスタ

| ID | 要件 |
|----|------|
| F-10 | マスタは **摘要キーワード**（摘要への部分一致）→ **自社PL勘定項目**（ドロップダウン）。 |
| F-11 | **金額下限・上限** 任意。ソフトバンク等の金額帯分離に利用。 |
| F-12 | **データソース区分** 任意。マスタ側にソースを書き、取引側と一致時のみ適用する絞り込みが可能。 |
| F-13 | 複数ルールが重なる場合、**キーワードが長いもの優先**、次に **金額レンジ一致** を優先（仕様書参照）。 |
| F-14 | 初期マスタは **手打ちPL運用** を反映したキーワード例を格納（`sample_master.csv`）。 |

### 3.3 支給控除・人件費

| ID | 要件 |
|----|------|
| F-20 | ジョン様形式「支給控除一覧表（部門別）」**xlsx/csv** を読み、指定行ラベルの合計を **人件費(支給額,健康,介護,厚生,子ども)** とする。 |
| F-21 | 対象列は **表の1行目**（氏名・列見出し行）から **キーワード**（**本部・桜木町・新子安・白根・さいわい** を UI で複数選択、既定は全選択）で列マッチ。 |

### 3.4 共通・除外

| ID | 要件 |
|----|------|
| F-30 | 摘要に **「オリコ」** を含む行を振分対象から除外できる（既定オン）。 |
| F-31 | データソース自動付与（列が無いとき）: プリセットに応じて あおぞら／アメックス／横浜信金／エネクスフリート。 |

### 3.5 自社PL勘定項目

| ID | 要件 |
|----|------|
| F-40 | 要件定義書 v2 の 2.2 をベースに一覧を保持。収入・販管費・集計用項目を含む。 |
| F-41 | 支給控除集計の合計名 **人件費(支給額,健康,介護,厚生,子ども)** をプルダウンに含める。 |

## 4. 非機能要件

| ID | 内容 |
|----|------|
| NF-01 | 実行環境: Python 3.x、主要依存は `requirements.txt` に記載。 |
| NF-02 | PDF: PyMuPDF（エネフリ・横浜スキャン）、OCR は EasyOCR または Tesseract（任意）。 |
| NF-03 | スキャンOCR結果は **100%正確でない** 前提とし、要確認・不明を必ず区別可能とする。 |

## 6. 用語

| 用語 | 説明 |
|------|------|
| 確定（振分） | マスタに照らしキーワード・金額が整合した状態 |
| 要確認（振分） | キーワードはあるが金額レンジ外など |
| 判断不能 | マスタに一致するキーワードなし |
| 取込ステータス（横浜OCR） | OCR行の信頼度に基づく確定／要確認／不明（CSV振分の「確定」とは別概念） |
